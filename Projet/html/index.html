Projet de CDAA : Logiciel de gestion de relations clients (CRM)

Introduction

Le sujet du projet est de produire un logiciel permettant de gérer des contacts et leurs interactions avec une entité. Il est également question d'extraire des tâches a faire des interactions grâce â des tags (@ todo et @ date). Nous avons utilisé Github pour collaborer: https://github.com/Dwarron/ProjetCDAA.

Programmation en 3 parties

La programmation du projet est realisée en 3 parties distinctes : la programmation orientée object, l'interface graphique et la persistence des données.

Partie Programmation Orientée Object

Dans cette premiere partie, le but a ete de simplement décrire les differents objets qui interviendront tout le long du projet et les relations/interactions entre eux. Nous avons notamment crée les classes Contact, Verificator, GestionContact, Interaction et Todo. Ces classes ayant besoin ,de gerer le temps, nous avons également produit la classe Date.

Partie Interface Graphique

Dans cette seconde partie, nous avons développé differente interfaces permettant aux utilisateurs d'interagir avec notre application. Ces fen$egrave;etres sont gérées par les classes AfficheInteractionsWindow, AjoutInteractionWindow, CreationFicheWindow, FicheContactWindow, MainWindow, RechercheContactWindow et RequeteWindow.

Partie Persistence des données

Dans cette troisime et derniére partie, une base de données SQLite a été mise en place afin de garder en mémoire les données entrées par l'utilisateur. Nous utilisons la classe DatabaseManager pour interagir avec celle ci.
